================================================================================
STOCK PLATFORM DEPLOYMENT GUIDE
================================================================================

This guide walks you through deploying your stock platform with:
- Backend on Railway (with PostgreSQL)
- Public Frontend on Vercel
- Admin Frontend on Vercel (separate deployment)

================================================================================
OVERVIEW
================================================================================

Your platform is now split into:

1. BACKEND (Single Instance)
   - FastAPI app with all endpoints
   - Public endpoints (no auth)
   - Admin endpoints (require API key)
   - Background schedulers run automatically

2. PUBLIC FRONTEND
   - Shows: Dashboards, Momentum, News, Fundamentals
   - Hides: Debug tab
   - No admin API key

3. ADMIN FRONTEND  
   - Shows: All tabs including Debug
   - Has admin API key for authenticated requests
   - Same codebase, different environment variables

================================================================================
STEP 1: GENERATE ADMIN API KEY
================================================================================

Generate a strong random API key that will be shared between backend and admin frontend:

# Option 1: Using Python
python3 -c "import secrets; print(secrets.token_urlsafe(32))"

# Option 2: Using OpenSSL
openssl rand -base64 32

# Option 3: Using online generator
https://www.random.org/strings/?num=1&len=32&digits=on&upperalpha=on&loweralpha=on

Save this key! You'll need it for both backend and admin frontend.

================================================================================
STEP 2: DEPLOY BACKEND ON RAILWAY
================================================================================

1. Go to https://railway.app

2. Create New Project → Deploy from GitHub repo

3. Select your repository → Choose "backend" directory

4. Railway will auto-detect Dockerfile

5. Add PostgreSQL database:
   - Click "New" → Database → PostgreSQL
   - Railway automatically creates DATABASE_URL

6. Set Environment Variables (in Railway dashboard):
   
   DATABASE_URL                    (auto-generated by Railway)
   GOOGLE_API_KEY                  your_gemini_api_key
   ALPHA_VANTAGE_API_KEY           your_alpha_vantage_key
   ALPHAVANTAGE_API_KEY            your_alpha_vantage_key
   FRED_API_KEY                    your_fred_api_key
   BEA_API_KEY                     your_bea_api_key
   FMP_API_KEY                     your_fmp_api_key (or "demo")
   ADMIN_API_KEY                   <the key you generated in Step 1>
   DEBUG                           False
   CORS_ORIGINS                    ["https://yoursite.com","https://admin.yoursite.com"]
   LLM_MODEL                       gemini-2.5-flash

7. Deploy!
   - Railway will build and deploy automatically
   - Get your backend URL: something like https://your-app.up.railway.app

8. Add Custom Domain (Optional):
   - Go to Settings → Domains
   - Add domain: api.yoursite.com
   - Configure DNS records as instructed by Railway

9. Run Database Migrations:
   - In Railway dashboard, go to your service
   - Click "..." → Shell
   - Run: alembic upgrade head

================================================================================
STEP 3: DEPLOY PUBLIC FRONTEND ON VERCEL
================================================================================

1. Go to https://vercel.com

2. New Project → Import Git Repository

3. Select your repository

4. Configure Project:
   - Framework Preset: Next.js
   - Root Directory: frontend
   - Build Command: npm run build
   - Output Directory: .next

5. Set Environment Variables:
   
   NEXT_PUBLIC_API_URL             https://your-backend.up.railway.app
   NEXT_PUBLIC_ADMIN_MODE          false

   NOTE: DO NOT add ADMIN_API_KEY here!

6. Deploy!
   - Vercel will build and deploy
   - You'll get URL: https://your-project.vercel.app

7. Add Custom Domain:
   - Go to Project Settings → Domains
   - Add domain: yoursite.com
   - Configure DNS as instructed

================================================================================
STEP 4: DEPLOY ADMIN FRONTEND ON VERCEL
================================================================================

1. In Vercel, create another New Project

2. Import same Git Repository

3. Configure Project:
   - Framework Preset: Next.js
   - Root Directory: frontend  (same as public!)
   - Build Command: npm run build
   - Output Directory: .next

4. Set Environment Variables (DIFFERENT from public):
   
   NEXT_PUBLIC_API_URL             https://your-backend.up.railway.app
   NEXT_PUBLIC_ADMIN_MODE          true
   NEXT_PUBLIC_ADMIN_API_KEY       <the same key you used for backend>

   IMPORTANT: ADMIN_API_KEY must match exactly with backend!

5. Deploy!
   - You'll get URL: https://your-admin-project.vercel.app

6. Add Custom Domain:
   - Go to Project Settings → Domains  
   - Add domain: admin.yoursite.com
   - Configure DNS

7. Optional: Add Password Protection
   - Go to Project Settings → Password Protection
   - Enable password for extra security layer

================================================================================
STEP 5: VERIFY DEPLOYMENT
================================================================================

1. Test Backend Health:
   Visit: https://your-backend.up.railway.app/health
   Should return: {"status": "healthy", ...}

2. Test Public Frontend:
   - Visit: https://yoursite.com
   - Should see: Dashboards, Momentum, News, Fundamentals tabs
   - Should NOT see: Debug tab
   - Test stock search

3. Test Admin Frontend:
   - Visit: https://admin.yoursite.com
   - Should see: All tabs including Debug
   - Go to Debug tab
   - Try "Force Refresh" buttons
   - Should work without 401/403 errors

4. Test Admin Endpoints Directly:
   # Should fail (no auth)
   curl https://your-backend.up.railway.app/api/debug/gemini-calls
   
   # Should succeed (with auth)
   curl -H "X-Admin-API-Key: your_key_here" \
        https://your-backend.up.railway.app/api/debug/gemini-calls

================================================================================
TROUBLESHOOTING
================================================================================

Problem: Admin endpoints return 401 Unauthorized
Solution: Check that ADMIN_API_KEY matches in backend and admin frontend

Problem: Debug tab doesn't show in admin
Solution: Verify NEXT_PUBLIC_ADMIN_MODE=true in admin frontend

Problem: CORS errors
Solution: Update CORS_ORIGINS in backend to include both frontend domains

Problem: Database connection fails
Solution: Check DATABASE_URL is set correctly in Railway

Problem: Schedulers not running
Solution: Check Railway backend logs, schedulers start automatically on deploy

Problem: Frontend can't reach backend
Solution: Check NEXT_PUBLIC_API_URL points to correct backend URL

================================================================================
UPDATING YOUR DEPLOYMENT
================================================================================

Backend Updates:
1. Push changes to GitHub
2. Railway auto-deploys from main branch
3. Check deployment logs in Railway dashboard

Frontend Updates:
1. Push changes to GitHub
2. Both Vercel projects auto-deploy
3. No need to change environment variables

Environment Variable Changes:
1. Railway: Update in dashboard, redeploy
2. Vercel: Update in project settings, redeploy

✓ DEBUG=False in production backend
✓ CORS_ORIGINS only includes your domains
✓ Public frontend has ADMIN_MODE=false
✓ Admin frontend doesn't have public domain
✓ Optional: Enable Vercel password protection on admin

✓ All API keys are in environment variables, not code
